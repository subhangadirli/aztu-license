# AzTU License v2.1 — FAQ

---

## ❓ General Questions

### What is the AzTU License?

AzTU License v2.1 is a strong copyleft open source license designed to ensure that software **always remains open source**.

It is inspired by the principles of the GNU Affero General Public License (AGPL), with a clearer and more explicit focus on perpetual openness.

---

### Why was this license created?

This license was created to:

* Make strong copyleft easier to understand
* Explicitly prevent software from becoming closed source
* Ensure source code availability in all scenarios, including SaaS
* Provide a simpler alternative to complex legal licenses

---

### Is this an official open source license?

No.

* It is **not approved** by the Open Source Initiative (OSI)
* It has **not been formally reviewed** by a legal professional

Use at your own risk.

---

## ⚖️ Legal & Usage

### Can I use this license for commercial projects?

✅ Yes.

You can:

* Sell the software
* Use it in commercial environments
* Offer paid services

❗ However:

* You must provide full source code
* You cannot make the software proprietary

---

### Can I make a closed-source version?

❌ No.

This is strictly prohibited.

All derivative works must remain open source and accessible.

---

### Do I have to share my source code?

✅ Yes.

You must provide full source code if you:

* Distribute the software
* Modify the software
* Use it to provide services over a network (e.g., SaaS, APIs)

---

### What counts as "source code"?

Source code means:

* The complete, human-readable code
* Build scripts and configuration files
* Everything required to modify and rebuild the software

---

### Can I use this in a SaaS product?

✅ Yes.

But:

* You must provide users with access to the full source code
* This applies even if you do not distribute the software directly

---

### Can I combine this with other open source licenses?

⚠️ It depends.

* Compatible with strong copyleft licenses like AGPL (in principle)
* May NOT be compatible with permissive licenses in all cases

If unsure, consult a legal expert.

---

### Can I relicense the software?

❌ No (in general).

You cannot relicense the software under a proprietary license.

Any redistribution must preserve the same freedoms.

## 🧠 Philosophy

### How is this different from AGPL?

The GNU Affero General Public License (AGPL):

* Is legally battle-tested
* Has strong copyleft and SaaS protection

AzTU License:

* Focuses on clarity and simplicity
* Explicitly states the “never closed source” principle
* Is easier to read and understand

However:

* It is not a replacement for AGPL
* It relies on similar principles but is independently written

---

### Why not just use AGPL?

You should use AGPL if you want:

* A legally proven license
* Broad industry acceptance

Use AzTU License if you want:

* Clear philosophy and messaging
* Explicit enforcement of perpetual openness
* Simpler structure
