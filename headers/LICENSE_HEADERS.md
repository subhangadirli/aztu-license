### SPDX Identifier Guide
Because AzTU is an emerging, non-OSI-approved license, it does not yet have an official, globally recognized SPDX short identifier. 

When using SPDX tags in your package managers (like `package.json` or `Cargo.toml`), use the standard `LicenseRef-` prefix designated for custom licenses:

`SPDX-License-Identifier: LicenseRef-AzTU-2.1`
